# 커밋 컨벤션

커밋 메시지는 아래 형식을 사용합니다.

```text
type: 변경 내용 요약
```

예시:

```text
feat: 코스 상세 화면 추가
fix: GPS 거리 계산 오류 수정
docs: Windows 실행 방법 정리
```

## 타입

| 타입 | 사용 상황 | 예시 |
| --- | --- | --- |
| `feat` | 새로운 기능 추가 | `feat: 러닝 기록 화면 추가` |
| `fix` | 버그 수정 | `fix: 거리 소수점 표시 오류 수정` |
| `docs` | 문서 수정 | `docs: PR 규칙 추가` |
| `style` | 포맷팅, 세미콜론 등 동작과 무관한 수정 | `style: 코드 포맷 정리` |
| `refactor` | 기능 변화 없는 구조 개선 | `refactor: 코스 카드 컴포넌트 분리` |
| `test` | 테스트 추가 또는 수정 | `test: 코스 API 테스트 추가` |
| `chore` | 설정, 패키지, 빌드 관련 수정 | `chore: ESLint 설정 추가` |
| `remove` | 파일 또는 코드 삭제 | `remove: 사용하지 않는 이미지 삭제` |
| `rename` | 파일명 또는 폴더명 변경 | `rename: RunPage 파일명 변경` |

## 작성 규칙

- 제목은 50자 안팎으로 짧게 작성합니다.
- 제목 끝에 마침표를 붙이지 않습니다.
- 한 커밋에는 하나의 목적만 담습니다.
- 여러 기능을 한 번에 묶어 커밋하지 않습니다.

## 좋은 예시

```text
feat: 코스 필터에 구 단위 선택 추가
fix: 완료 화면에서 평균 페이스 표시 수정
docs: Expo 실행 방법 추가
chore: 모바일 lint 스크립트 추가
```

## 피해야 할 예시

```text
수정
작업함
feat: 이것저것 추가
fix: 버그 수정하고 화면도 추가하고 문서도 고침
```
